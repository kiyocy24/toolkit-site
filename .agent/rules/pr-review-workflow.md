# PR Review Workflow (Gemini Code Assist)

Pull Request (PR) を作成した後、`gemini-code-assist` からの自動レビューを待機し、適切に対応するためのワークフローです。

## 1. レビュー待機
PR を作成したら、すぐに次のタスクには移らず、`gemini-code-assist` からのレビューコメントが付くのを待機してください。

## 2. コメントへの対応
レビューコメントが付いたら、その内容を確認します。

### A. 指摘が妥当な場合
1. コードの修正を行います。
2. 変更をコミットし、リモートブランチに `push` します。
3. `push` 後、該当するレビューコメントに対して修正内容を説明するリプライを返し、「Resolve conversation」ボタンでコメントを解決済みにしてください。（例：「修正しました」「対応しました」）

### B. 指摘が的を射ていない（不適切）場合
1. コードの修正は行いません。
2. レビューコメントに対して、なぜその指摘が適切でないか、または意図的な実装であるかを説明する反論のリプライを返してください。

## 3. 人間による最終レビュー
全てのコメントに対応（修正または反論）したら、人間のレビュアーにレビューを依頼してください。AIの指摘への対応が適切であるかを含め、最終的な承認は人間のレビュアーが行います。
