# AGENTS.md

このファイルは、AIコーディングエージェント（Gemini 3 Pro等）がこのプロジェクトにおいて自律的にタスクを遂行するための「コンテキストとルールの定義書」です。
AIエージェントは、コードの変更、ファイルの作成、リファクタリングを行う際、**必ずこのファイルの指示に従ってください。**

## 1. プロジェクト概要
このプロジェクトは、**Nextra (Next.jsベース)** を使用したドキュメント/静的ウェブサイトです。
Markdown/MDXファイルをベースに、高速かつ高機能なドキュメントサイトを構築しています。

## 2. 技術スタック & 環境
AIエージェントは以下の技術スタックを厳守し、それ以外の代替技術（npm, yarn, pure CSSなど）を提案・使用しないでください。

- **Framework:** Nextra (Next.js)
- **Language:** TypeScript (`.ts`, `.tsx`, `.mdx`)
- **Package Manager:** pnpm
- **Styling:** Tailwind CSS
- **Deployment:** Vercel (想定)

## 3. 開発ルールとコマンド
すべてのコマンド実行は **pnpm** を使用してください。

- **依存関係のインストール:** `pnpm install`
- **開発サーバー起動:** `pnpm dev`
- **ビルド:** `pnpm build`
- **Lint/Format:** プロジェクト設定に従う（Prettier/ESLint推奨）

## 4. ディレクトリ構造とルーティング (Nextra特有)
Nextraは**ファイルシステムベースのルーティング**を採用しています。AIエージェントは以下の構造を理解し、適切な場所にファイルを配置してください。

- `/pages`: ページコンテンツの格納場所。
  - `_meta.json`: サイドバーやナビゲーションの構造を定義する重要なファイル。新しいページを追加した際は、必ず同階層の `_meta.json` も更新すること。
  - `.md` / `.mdx`: 各ページのコンテンツ。Reactコンポーネントを使用する場合は必ず `.mdx` を使用すること。
- `/components`: 共通Reactコンポーネント（TypeScriptで記述）。
- `/theme.config.tsx`: Nextraのテーマ設定ファイル（ロゴ、フッター、SEO設定など）。

## 5. コーディング規約

### 5.1 TypeScript
- **Strict Mode:** 厳格な型チェックを行うこと。`any` 型の使用は極力避け、適切なインターフェースや型定義を行うこと。
- **Component:** 関数コンポーネント（FC）を使用し、Propsの型定義を明示すること。

### 5.2 Tailwind CSS
- スタイリングは原則として Tailwind CSS のユーティリティクラスを使用すること。
- 複雑なスタイルや再利用可能なスタイルは、`@apply` ディレクティブやカスタムコンポーネント化を検討すること。
- Nextraのデフォルトテーマと競合しないよう注意すること。

### 5.3 Nextra / MDX
- 記事内のインタラクティブな要素は、カスタムReactコンポーネントとして作成し、MDX内でインポートして使用すること。
- リンクはNext.jsの `<Link>` コンポーネントではなく、Markdown標準の `[リンクテキスト](URL)` またはNextraが処理可能な形式を使用すること。

## 6. AIエージェントへの具体的な指示 (Behavior Instructions)

1.  **`_meta.json` の更新を忘れないこと:**
    新しいページファイル（例: `about.mdx`）を作成した場合は、必ずそのディレクトリの `_meta.json` にエントリを追加し、ナビゲーションに表示されるようにしてください。

2.  **MDXの活用:**
    「リッチなコンテンツ」や「動的な要素」を求められた場合は、`.md` ではなく `.mdx` を選択し、Reactコンポーネントを組み込んで実装してください。

3.  **画像の扱い:**
    静的画像は `/public` ディレクトリに配置し、MDXからは `![alt](/image.png)` の形式で参照してください。Next.jsの `Image` コンポーネントによる最適化が必要な場合は、Nextraの仕様に従って実装してください。

4.  **回答のトーン:**
    コード生成時のコメントやコミットメッセージは日本語で記述してください。
